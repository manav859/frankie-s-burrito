# Prompt Improver

Use this when a prompt needs to produce more reliable structured output.

Workflow:
1. Define the target outcome, inputs, and output format.
2. Remove vague adjectives and replace them with concrete instructions.
3. Add constraints, failure handling, and explicit schema when needed.
4. Include a small high-quality example only if it materially improves accuracy.
5. Optimize for consistency before cleverness.

Rules:
- A good prompt states role, task, context, constraints, and output shape.
- If you need structured output, specify the structure directly.
- Prompt quality is measured by downstream reliability, not by sounding elaborate.
