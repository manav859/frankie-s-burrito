# Migration And Redirects

Use this when moving content, changing URL structures, or changing delivery architecture.

Workflow:
1. Inventory current URLs, content entities, and inbound dependencies.
2. Define the target URL strategy and canonical behavior.
3. Build explicit redirect rules for changed or removed paths.
4. Validate analytics, sitemap, internal links, and schema against the new structure.
5. Plan rollback, monitoring, and post-launch verification.

Rules:
- Migrations fail when URL mapping is treated as cleanup work.
- Prefer permanent redirects for durable moves and keep the map reviewable.
- Preserve search equity, bookmarks, and editor workflows during cutover.
